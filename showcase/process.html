// Key Decisions

开发过程中的关键决策

AI 辅助开发中的取舍与判断

选择 Phaser 而非纯 DOM

像素游戏需要精确的 Canvas 渲染控制和精灵管理，Phaser 提供成熟的 2D 游戏框架。

✓ 渲染性能优秀

暗色底 + 金色强调的色板

参考 Downwell 的极简配色，用深蓝黑做底、金色做强调色，营造"天生骄傲"的视觉基调。

✓ 视觉辨识度高

角色立绘而非纯像素小人

放弃传统 16x16 像素小人，改用高质量像素风立绘，视觉小说风格更有代入感。

✓ 角色表现力强

DOM + Canvas 混合渲染

UI 层用 DOM overlay 保证文字清晰和中文字体渲染，游戏层用 Canvas 保证动画流畅。

✓ 中文显示完美
// Lessons Learned

创作心得

  • 1 AI 是超级加速器，但方向感要自己把握。 TRAE 能快速实现技术方案，但游戏的设计哲学 —— "不评判选择" —— 这种核心理念必须由人来定义和坚守。
  • 2 模块化是 AI 协作的关键。 把系统拆成独立的模块（PixelRenderer、DialogSystem、ChoiceSystem），每次让 TRAE 聚焦一个模块开发，效果远好于一次性生成整个项目。
  • 3 像素美术不是简陋，是克制。 暗色调 + 金色点缀 + 扫描线，用最少的视觉元素传达最强的氛围。Less is more 在像素风格里尤其成立。
  • 4 自动化测试不可或缺。 70+ 剧情节点手动测试不现实。用 TRAE 生成的 E2E 脚本自动走查所有分支，确保没有死链和错字。
  • 5 有感而发 > 追求完美。 这个项目的出发点不是"做一个好玩的游戏"，而是有感而发。选择没有对错，做出来就是最好的答案。
