# 工作者提示词编写指南 v2

## 核心原则

> 先说问题，再说方案。工作者只有理解了"为什么"，才能做对"做什么"。

---

## 提示词结构（按顺序）

### 1. 项目背景（让工作者理解全局）

```
这是一个什么项目 → 技术栈是什么 → 设计风格是什么 → 当前处于什么阶段
```

### 2. 当前问题（让工作者理解为什么要做）

```
现在是什么样 → 有什么具体问题 → 问题导致了什么体验缺陷
```

**关键：用事实说话，不用模糊形容词。**

❌ "对话框不够好看"
✅ "对话框目前是纯黑底+2px金色边框，没有任何装饰元素，看起来像开发者调试界面而不是像素风游戏UI。在800×500的画布上，对话框占据了底部1/3的面积，但视觉上毫无存在感。"

### 3. 目标状态（让工作者理解做完应该什么样）

```
改完之后是什么样 → 用户看到什么变化 → 和项目整体风格怎么统一
```

**关键：描述最终效果，而不是实现步骤。**

❌ "加像素风装饰角"
✅ "对话框应该看起来像复古像素游戏的对话面板——四角有金色L形装饰，背景有从深到浅的微妙渐变，角色名下方有金色下划线，整体和标题界面的像素风按钮风格一致。"

### 4. 具体要求（让工作者知道边界）

```
必须做到什么 → 不能做什么 → 哪些文件可以改 → 哪些文件不能碰
```

### 5. 依赖关系（让工作者知道和谁协作）

```
你改的文件被谁调用 → 谁同时在改相关文件 → 执行顺序
```

### 6. 验证标准（让工作者知道怎么判断做对了）

```
功能验证 → 视觉验证 → 构建验证
```

## 对比示例

### ❌ 差的提示词

```
你的任务是优化 DialogSystem.js，让对话框更有像素风质感。
需要加装饰角、渐变背景、闪烁动画。字体换成 Press Start 2P。
```

**问题**：
- 没说现在对话框长什么样 → 工作者不知道从哪改起
- 没说为什么要改 → 工作者不理解设计意图
- 没说改完应该什么样 → 工作者只能猜测"像素风"是什么意思
- 没说不能改什么 → 可能改坏其他系统

### ✅ 好的提示词

```
## 项目背景
这是一个以罗永浩人生经历为背景的 2D 像素风互动视觉小说游戏。
技术栈：Phaser 3.80 + Vite 5，纯 Canvas 渲染。
设计风格：所有 UI 元素用代码绘制（fillRect），复古像素风，
不使用任何图片资源。色调以暗色为底、金色（#f0c040）为强调色。

## 当前问题
DialogSystem.js 中的对话框存在以下问题：

1. **视觉上像调试界面**：对话框是纯黑底（0x0a0a1a）+ 2px 金色边框，
   没有任何装饰元素。在 800×500 画布上占据底部 140px 高度，
   但看起来像开发者工具而不是游戏 UI。

2. **角色名没有视觉层次**：角色名和正文用同样的排版方式，
   没有视觉区分，玩家不容易快速识别"谁在说话"。

3. **"点击继续"提示没有动画**：提示文字是静态的，
   玩家可能注意不到需要点击才能继续，导致卡住。

4. **字体不统一**：所有文字用系统 monospace 字体，
   和标题界面的 Press Start 2P 像素字体风格不一致，
   破坏了像素风的沉浸感。

## 目标状态
改完后，对话框应该：

1. **看起来像复古像素游戏的对话面板**：四角有金色 L 形装饰角
   （和标题界面按钮的装饰角风格一致），背景有从深到浅的微妙渐变，
   整体有"游戏UI"的质感而不是"调试框"。

2. **角色名有视觉层次**：名字下方有金色下划线，
   名字用 Press Start 2P 像素字体（和标题统一），
   正文用 Noto Sans SC（保证中文可读性）。

3. **"点击继续"有呼吸闪烁**：alpha 在 0.3-1 之间循环，
   引导玩家点击。

4. **打字时有微弱音效**：每 3 个字符播放一次极短的"嘟"声
   （800Hz, 0.02s, square wave, volume 0.03），
   增强打字机的沉浸感。

## 你要修改的文件
路径：e:\ownWorkPlace\罗的十字路口\luohammer-pixel-game\src\systems\DialogSystem.js

先 Read 这个文件，理解当前代码结构后再修改。
关键结构：
- constructor：创建对话框 UI 元素（bg, nameText, storyText, hintText, hitZone）
- show()：显示对话框，启动打字机效果
- typeNext()：逐字显示文本
- hide()：隐藏对话框

## 具体要求

### 必须做到
1. 对话框四角加 L 形金色装饰角（每个角 2 个 fillRect，8×8 像素）
2. 背景从纯黑改为 4 行渐变（0x0a0a1a → 0x12122a）
3. 角色名下方加 2px 金色下划线
4. "点击继续"加 alpha 闪烁动画（用 scene.tweens）
5. 打字时每 3 个字符播放一次音效（用 scene.audio._playTone）
6. 角色名字体改为 Press Start 2P 10px
7. 正文字体改为 Noto Sans SC 13px
8. 提示字体改为 Press Start 2P 8px

### 不能做
1. 不要修改 show/hide/typeNext 的方法签名
2. 不要修改 hitZone 的交互逻辑
3. 不要修改 container 的 depth（10）
4. 不要引入新的 import
5. 不要修改 ChoiceSystem.js（另一个工作者在改）

### 依赖关系
- GameScene.js 调用 this.dialog.show(charInfo[0], node.text, callback)
- GameScene.js 调用 this.dialog.hide()
- AudioSystem 在 GameScene.create() 中初始化为 this.audio
- 所以在 show() 方法中用 this.scene.audio 获取引用（延迟获取）

## 验证标准
1. npm run build 无报错
2. 对话框四角能看到金色 L 形装饰
3. 角色名下方有金色线
4. "点击继续"有闪烁效果
5. 打字时有微弱音效
6. 字体风格和标题界面一致
```

## 提示词自检清单

发送前逐项检查：

### 问题是否说清楚了？
- [ ] 我说了"现在是什么样"吗？（用代码/截图/数据描述）
- [ ] 我说了"有什么问题"吗？（具体的、可观测的）
- [ ] 我说了"问题导致什么后果"吗？（用户体验/功能/一致性）

### 目标是否说清楚了？
- [ ] 我说了"改完应该什么样"吗？（描述最终效果）
- [ ] 我说了"和项目整体风格怎么统一"吗？
- [ ] 我给了参考吗？（同项目中已有的、风格一致的示例）

### 约束是否说清楚了？
- [ ] 我说了"必须做到什么"吗？
- [ ] 我说了"不能做什么"吗？
- [ ] 我说了"哪些文件可以改/不能碰"吗？

### 依赖是否说清楚了？
- [ ] 我说了"谁调用你改的代码"吗？
- [ ] 我说了"谁同时在改相关文件"吗？

### 验证是否说清楚了？
- [ ] 我说了"怎么判断做对了"吗？
- [ ] 我说了"构建必须通过"吗？

## 常见错误与修正

| 错误写法 | 问题 | 正确写法 |
|---|---|---|
| "优化UI" | 太模糊，工作者不知道优化什么 | "对话框目前是纯黑底+2px边框，看起来像调试界面，需要改成像素风游戏对话面板" |
| "加像素风装饰" | 没说现在什么样，没说目标什么样 | "当前对话框四角是空的，需要加金色L形装饰角，和标题界面按钮的装饰角风格一致" |
| "让角色更有辨识度" | 没说当前角色长什么样 | "当前角色是12px宽的通用小人，没有黑框眼镜等特征，在800×500画布上几乎看不清。需要放大到24px+，加上黑框眼镜" |
| "参考项目风格" | 没给具体参考 | "参考 BootScene.js 中 createButton 的像素风按钮风格：深蓝底+金色边框+L形装饰角" |
| "不要改其他文件" | 太笼统 | "不要修改 GameScene.js（另一个工作者在改），不要修改 config.js（颜色配置由 W4 负责）" |
