# W1-b：基础场景分层优化

## 项目背景
这是一个以罗永浩人生经历为背景的 2D 像素风互动视觉小说游戏。
技术栈：Phaser 3.80 + Vite 5，纯 Canvas 渲染（graphics.fillRect）。
画布尺寸：800×500。
项目根目录：`e:\ownWorkPlace\罗的十字路口\luohammer-pixel-game`

## 当前问题
6 个基础场景只有墙面+地面两层，没有前景/中景/远景分层，看起来像色块拼接而不是游戏场景。

## 参考图和规范
- 之前豆包生成的 10 张场景图（教室/演讲厅/办公室/发布会/直播间/街道）
- `.trae/documents/visual-design-spec.md` — 色板

## 你要修改的文件
`src/systems/PixelRenderer.js` — 只修改 6 个基础场景方法

### 绘制代码结构
每个方法内部按这个顺序绘制（后面的覆盖前面的）：
```javascript
drawClassroom(g) {
  // 1. 背景（天空/墙壁上部）
  g.fillStyle(c.wall); g.fillRect(...);
  // 2. 中景（主要物体）
  g.fillStyle(c.floor); g.fillRect(...);
  // 3. 前景（近处物体，不遮挡角色头部）
  g.fillStyle(c.detail); g.fillRect(...);
}
```

3. 不要修改 config.js

## 验证标准
1. `npm run build` 无报错
2. 每个场景有明显的 3 层深度（背景/中景/前景）
3. 前景元素不遮挡角色头部区域（y < 250）
4. 场景色调和参考图一致
5. 所有 fillRect 尺寸是 4 的倍数
