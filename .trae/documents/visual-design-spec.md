# 真还传 · 视觉设计规范（Visual Design Spec）

## 核心原则

> 本项目的像素风 = **暗色赛博 + 复古 8-bit + 中国互联网文化**
> 不是 FC 红白机，不是日式 RPG，不是 Undertale。
> 参考：如果《赛博朋克 2077》做成 8-bit 像素风，大概就是这个感觉。

---

## 一、参考游戏（给模型/设计师看）

| 游戏 | 我们借鉴什么 | 不借鉴什么 |
|---|---|---|
| **Celeste** | 精致的像素角色、流畅的动画帧 | 太日式、太可爱 |
| **Downwell** | 极简配色、强对比 | 太简陋 |
| **Hyper Light Drifter** | 暗色氛围、场景层次 | 太抽象、角色太小 |
| **Vampire Survivors** | 角色辨识度、UI 简洁度 | 太粗糙 |
| **Papers, Please** | 叙事 UI、严肃氛围 | 太灰暗 |
| **Disco Elysium** | 对话系统 UI、选择设计 | 不是像素风 |

**最接近我们目标的视觉风格**：暗色底 + 金色强调 + 像素风角色 + CRT 扫描线质感

## 三、像素网格规范

### 基本单位
- **网格单位 G = 4px**（所有元素尺寸必须是 4 的倍数）
- **画布：800×500**
- **角色高度：约 60-70px**（占画布高度的 12-14%）
- **角色宽度：约 24-32px**

### 像素风规则
1. **所有 fillRect 的宽高必须是 4 的倍数**
2. **线条宽度必须是 2px 或 4px**（不用 1px，太细看不清）
3. **颜色不用渐变**（纯色块，用不同色阶模拟明暗）
4. **角色轮廓用 2px 深色线**（#0A0A0A）
5. **文字用像素字体**（Press Start 2P / Noto Sans SC）

## 四、角色设计规范（老罗）

### 标志性特征（按重要性排序）

1. **黑框眼镜** — 最关键！必须一眼看到
   - 镜框：2px 粗的深色矩形
   - 镜片：深色半透明 + 白色 2×2 反光点
   - 鼻梁：2px 连接线

2. **圆脸/微胖** — 头部比身体宽
   - 头宽：24px，头高：20px
   - 用圆角矩形模拟（像素风的"圆"就是切角）

3. **寸头** — 不遮耳朵
   - 头顶 4px 深色区域
   - 两侧露出耳朵（肤色 4×4 方块）

4. **黑T恤** — 上半身深色
   - 领口 V 形（2px 深色缺口）

## 七、动画规范

| 动画 | 时长 | 缓动 | 说明 |
|---|---|---|---|
| 标题逐字出现 | 每字 300ms | Back.Out | 淡入 + Y轴弹跳 |
| 选项滑入 | 200ms | Back.easeOut | 从下方+40滑入 |
| 属性条填充 | 1000ms | Cubic.Out | 从0到目标值 |
| 成就闪光 | 300ms | Linear | 白色竖条左→右 |
| 提示闪烁 | 2000ms | Sine.easeInOut | alpha 0.3↔1 |
| 转场 | 500ms | Linear | 淡入淡出 |
| 粒子飘落 | 8000-18000ms | Linear | 循环 |

## 八、如何使用这份规范

### 给 AI 模型
把这份规范作为 system prompt 的一部分，或者作为任务描述的"参考规范"段落。
关键：**不要只说"像素风"，要给色板+网格+参考图**。

### 给人类设计师
用 Piskel (piskelapp.com) 或 Aseprite 按照网格规范绘制角色，导出为 sprite sheet。
用 Figma 按照色板和字体规范设计 UI 组件。

### 给前端开发者
所有颜色从 config.js 的 COLORS/UI_COLORS 读取。
所有字体从 config.js 的 FONTS 读取。
所有尺寸是 GRID(4) 的倍数。
角色绘制参考第四节的像素网格。
