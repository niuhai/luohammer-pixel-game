# Project Pilot — 项目自动驾驶仪

本 Skill 用于在获得用户授权后，自动推进《真还传 · 创业模拟器》项目。一次只执行一个任务，执行完向用户报告结果并建议下一步。

## 触发条件

当用户输入以下任意词语时触发：
- 开工
- 继续
- 迭代
- 跑下一个
- 做下一个
- 开始调度
- 推进项目

## 核心工作流

```
1. 读取任务池 → 2. 选择首个 pending 任务 → 3. 理解任务 → 4. 执行
→ 5. 验证 → 6. 更新任务状态 → 7. 报告结果并建议下一步
```

### Step 1：读取任务池

读取 `docs/task-pool.md` 或 `.trae/skills/concurrent-dispatcher/task-pool.md`（以存在的为准）。

如果两个文件都不存在，读取 `docs/plans/` 下的最新计划文档作为任务源。

### Step 2：选择首个 pending 任务

- 找到第一个状态为 `pending` 的任务
- 如果没有 pending 任务，检查 `in_progress` 任务是否卡住
- 如果全部完成，恭喜用户并询问是否开启下一版本计划

### Step 3：理解任务

执行任务前必须：
1. 读取任务描述中列出的所有相关文件
2. 如果任务涉及新文件创建，先确认父目录存在
3. 如果任务涉及美术资源，记录需要生成/替换的资源清单

### Step 4：执行任务

根据任务类型选择执行策略：

#### 类型 A：代码任务
- 优先编辑现有文件，避免新建文件
- 修改前先完整读取目标文件
- 保持项目现有代码风格和命名规范
- 修改后运行 `npm run build` 验证

#### 类型 B：美术资源任务
- 检查是否已有 `worker-template.md` 或相关生成脚本
- 使用用户已有的图像生成资源（Seedream / Midjourney / SD 等）生成新资源
- 生成后压缩图片（目标：角色图 < 600KB，场景图 < 1MB）
- 更新 `src/config.js`、`src/scenes/GameScene.js` 中的资源引用

#### 类型 C：文案/数据任务
- 修改 `src/data/story.js` 或 `src/data/story/*.js`
- 保持数据格式一致，不破坏现有节点结构
- 修改后运行构建和简单剧情路径验证

### Step 5：验证

根据任务类型执行验证：

| 任务类型 | 必做验证 |
|---------|---------|
| 代码 | `npm run build` 通过 |
| UI/视觉 | 启动 dev server 并截图验证 |
| 数据 | 构建通过 + 关键路径手动验证 |
| 美术 | 检查图片加载正常、无水印、体积达标 |

### Step 6：更新任务状态

在任务池文件中将该任务状态更新为 `done`，并补充：
- 完成时间
- 主要修改文件
- 验证结果摘要

### Step 7：报告结果

向用户报告：
- 完成了哪个任务
- 修改了哪些文件
- 验证是否通过
- 是否引入了需要注意的问题
- 下一个 pending 任务是什么

报告格式示例：

```markdown
✅ 完成任务：T01 — 剧情可达性校验
- 修改文件：`src/data/story.js`, `scripts/validate-story.cjs`
- 验证：`npm run build` 通过，校验脚本 0 errors
- 说明：修复了 3 个死链、2 个孤立节点
- 下一个任务：T02 — 文案润色前半生
```

## 安全规则

1. **一次只执行一个任务**，不自动连续执行多个任务
2. **不自动执行 git commit / push**，除非用户明确说"提交"
3. **不删除用户已有文件**，除非任务明确要求
4. **不修改 `.env`、密钥、配置文件**等敏感内容
5. **执行前必须先读取相关文件**，禁止盲改
6. **验证失败时必须停止并报告**，不强行标记完成

## 与用户协作模式

- 任务执行过程中，如果需要用户做决策（如选 A 还是 B），暂停并提问
- 任务完成后，等用户说"继续"或"下一个"再推进
- 如果用户说"跳过"，将当前任务标记为 `skipped` 并移动到下一个
